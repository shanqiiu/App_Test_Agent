## 四、UI 界面描述关键点提取

### 4.1 首页 (Home)

**淘宝首页典型元素：**
- 顶部搜索框：常含默认热搜词（如"筋膜枪"、"华为p90"），右侧有搜索按钮
- 功能入口区域：百亿补贴、品牌精选等入口图标
- 推荐商品区域：卡片式展示，含图片、价格、销量
- 底部导航栏：首页、购物车、订单、我的

**京东首页典型元素：**
- 搜索框：默认内容（如"华为手机"、"华为p90"）
- 商品分类区：多个分类入口
- 促销信息区：秒杀、活动Banner
- 底部导航栏：首页、分类、购物车、我的

**拼多多首页典型元素：**
- 顶部搜索框
- 商品推荐区域
- 底部导航栏：首页、多多视频、购物车、我的

### 4.2 搜索结果页 (Search Results)

**共同元素：**
- 顶部搜索框：显示当前搜索关键词，可编辑
- 排序/筛选栏：综合、销量、价格排序等标签
- 筛选按钮：位于排序栏右侧，点击打开筛选面板
- 商品列表：卡片式布局，每个商品包含：
  - 商品图片
  - 商品名称/描述
  - 价格（红色高亮）
  - 销量数据（"已售xxx"）
  - 标签（百亿补贴/国补/京东自营等）

**各平台差异化元素：**

|------|---------|
| 淘宝 | "百亿补贴"标签、"国补"标识、天猫标识 |

### 4.3 筛选页 (Filter)

**典型筛选条件：**
- **价格区间**：固定档位（如"0-100"、"100-300"）+ 自定义区间（最低价/最高价输入框）
- **品牌**：多选checkbox
- **排序方式**：综合、销量、价格升序、价格降序

### 4.4 商品详情页 (Product Detail)

## 五、交互动作类型（Action Types）

| 动作类型 | 含义 | 使用场景 |
|---------|------|---------|
| `set_text(element, text)` | 输入文本 | 替换搜索框内容 |
| `click(element)` | 点击元素 | 点击排序标签、筛选按钮、商品卡片、规格选项等 |
| `click(element1)\nclick(element2)` | 连续点击 | 价格筛选：输入最高价 + 点确定 |
| `scroll(direction, search_expr, direction2)` | 滚动页面 | 在筛选页中滚动查找特定选项（如"国补"） |
| `clarify(question)` | 询问用户 | 用户意图不明确时请求确认（规格、支付方式等） |
| `do_nothing()` | 无操作/等待 | 页面加载中时等待加载完成 |
| `播报(content);\ndone(content)` | 播报结果并结束 | 任务完成，告知用户结果 |
| `scroll_until_found(direction, keyword, direction2)` | 滚动直到找到目标 | 在筛选页中搜索特定筛选项 |

**动作模式分布：**
| 模式 | 特点 |
|------|------|
| `fast_thinking` | 步骤更少、路径更直接、决策效率更高 |
| `7B` | 步骤更多、有更多回退和重试、交互更迂回 |

## 八、典型交互流程模式

### 模式 A：简单搜索购买（无筛选条件）
```
set_text(搜索词) → click(商品卡片) → clarify(规格) → click(购买) → done
```

### 模式 B：搜索 + 排序
```
set_text(搜索词) → click(排序标签) → done
```

### 模式 C：搜索 + 筛选 + 排序
```
set_text(搜索词) → click(筛选) → set_text(价格) → scroll(找选项) → click(完成)
→ click(排序标签) → done
```

### 模式 D：完整购买流程
```
set_text(搜索词) → click(筛选) → scroll → click(完成) → click(排序)
→ click(商品卡片) → click(购买) → clarify(规格) → click(确认)
→ clarify(支付方式) → click(支付) → done
```

### 模式 E：店铺内搜索
```
set_text(搜索词) → click(商品) → click(进店) → set_text(店内搜索)
→ click(价格筛选) → done
```

## 十四、关键发现

1. **步骤起点：** 所有任务的 stepId 均从 "3" 开始，前3步（0-2）为预记录的基础交互（如打开App、同意协议等）。

2. **fast_thinking vs 7B：**
   - `fast_thinking` 平均步骤数：~10步
   - `7B` 平均步骤数：~15步
   - `7B` 模式在复杂任务中出现更多回退操作

3. **常见失败原因：**
   - 筛选面板中价格输入框不可见（scroll 失效）
   - 用户意图模糊（需 clarify）
   - 页面加载延迟（需 do_nothing 等待）

4. **结束方式：** 约60% 以 `播报 + done` 结束任务，约40% 在 `clarify` 环节结束（因需要用户确认规格/支付方式等）。

5. **淘宝独有功能：** "百亿补贴"、"国补" 是淘宝特有的筛选条件，其他平台不涉及。

6. **京东独有标识：** "京东自营" 是最常见的筛选条件之一，用户高度重视。

7. **HuaweiShop（华为商城）特殊性：** 作为品牌自营商城，没有第三方卖家概念，所有商品均为官方直接销售，因此不涉及"店铺"和"自营"筛选。
