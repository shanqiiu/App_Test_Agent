# 研究路线图

AI智能体异常测试场景生成的整体研究规划和实施路线。

---

## 项目目标

构建一套完整的AI智能体异常测试场景自动生成平台，通过**正常行为采集—程序化异常生成—动态场景注入**三阶段流程，实现高仿真、上下文相关的异常场景批量生成。

---

```
         第一阶段                   第二阶段 🔥                第三阶段
```

## 研究阶段

## 第一阶段：正常行为采集 ⭐ 部分完成

**目标**: 系统性采集应用的正常交互行为，构建上下文基线

### 核心任务

#### 1. App页面自主遍历
- **方法**: Model-based Testing
- **工具**: Appium / Espresso
- **产出**: 完整的UI遍历路径
- **状态**: 未启动（当前阶段聚焦截图合成，不依赖运行时遍历）

#### 2. 界面状态结构化表征
- **视觉特征**: 截图、布局分析
- **结构特征**: DOM树、元素属性
- **语义特征**: 页面类型、功能标签
- **产出**: 结构化的UI状态数据
- **状态**: ✅ 通过 OmniParser + VLM 实现（产出 UI-JSON）

#### 3. UI交互图谱构建
- **状态**: 未启动（非当前阶段重点）

**相关文档**: [方案可行性分析 - 正常行为采集](../research/01_方案可行性分析.md#1-正常行为采集环节-)

## 第二阶段：程序化异常生成 🔥 当前阶段

**目标**: 基于截图和GT模板，生成高仿真的异常场景截图

### 核心任务

#### 1. 异常生成范式 ✅ 已实现
- **方案**: OmniParser粗检测 → VLM语义分组 → 程序化渲染
- **技术路线**: 确定性代码渲染 + DashScope API图像生成（非LoRA/Diffusion）
- **产出**: `ui_semantic_patch` 三阶段流水线

#### 2. 异常场景分类体系 ✅ 已建立
- **弹窗覆盖原UI** — 8个GT样本（`meta.json`驱动）
- **内容歧义、重复** — 底部浮层复制组件
- **loading_timeout** — 区域加载图标覆盖
- **局部文字覆盖** — 精确文本编辑

#### 3. 上下文感知合成 ✅ 已实现
- **meta-driven生成**: 读取GT模板的`meta.json`语义描述，驱动弹窗风格
- **VLM语义定位**: 根据指令关键词匹配UI-JSON组件，精确定位注入点
- **四种异常模式**: `dialog` / `area_loading` / `content_duplicate` / `text_overlay`

#### 4. 批量生成能力 ✅ 已实现
- **`batch_pipeline.py`**: 批量遍历原图目录 × GT类别

#### 5. 待完善
- [ ] 评估体系（保真度/上下文一致性量化指标）
- [ ] 更多GT模板类型（控件错位、文字乱码、色彩异常等）
- [ ] 风格一致性自动验证

**相关文档**: [程序化异常生成调研](../research/02_程序化异常生成调研.md)

## 风险与应对

### 技术风险

#### 1. 语义一致性难以保证
**风险**: 生成的异常不符合业务逻辑
**应对**: meta.json驱动精准风格控制 + VLM语义文案生成

#### 2. 生成质量不稳定
**风险**: AI图像生成结果不可控
**应对**: 确定性程序化渲染为主，DashScope AI为辅；中间结果全程保存便于调试

#### 3. GT模板覆盖不足
**风险**: 异常类型覆盖不全
**应对**: 持续扩充GT模板库，支持`generate_meta.py`自动生成meta描述

## 成功标准

### 技术指标

1. **覆盖率**: 生成的异常场景覆盖主要异常类型（目标10+类）
2. **保真度**: 生成界面与真实应用高度相似
3. **上下文相关性**: 异常注入符合页面语义
4. **多样性**: 同类型异常的视觉多样性

### 工程指标

1. **自动化程度**: 一条指令完成端到端生成
2. **生成效率**: 单张截图端到端 < 60秒
3. **批量能力**: 支持批量脚本自动化
