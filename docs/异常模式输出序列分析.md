# 异常模式输出序列分析

## 概述

本文档分析各异常渲染模式的输出产物及序列构造方式，说明在 `SequenceRewriter` 框架下各模式如何适配"注入前原图 → 异常图 → 恢复原图 → 后续步骤"的序列逻辑。

---

## 1. dialog 模式（弹窗覆盖）

### 渲染器
`PatchRenderer` (`app/renderers/patch.py`)

### 输出产物

| 文件 | 说明 | 是否进入序列 |
|---|---|---|
| `final_{timestamp}.png` | 原图 + 弹窗合成结果 | **是** |
| `dialog_only_{timestamp}.png` | 单独弹窗图像（不带背景） | 否（中间产物） |
| `vis_bbox_{timestamp}.png` | 关闭按钮检测框可视化 | 否（调试用） |

### 序列构造

```
step_N.jpg        - 原图（弹窗将出现于此页面）
step_N+1_anomaly  - 弹窗出现（异常图，基于 N 生成）
step_N+2.jpg      - 恢复（复制 N，关闭弹窗回到同一页面）
step_N+3.jpg      - 原图 N+1 平移（继续操作）
```

### 特殊处理

- 关闭按钮由 **PIL 代码绘制**（`patch.py:348-394`），AI 模型不参与
- 如 AI 自行绘制了关闭按钮，由 `_erase_ai_close_button` 后处理用背景色覆盖
- 输出只取 `final_*.png`，`dialog_only` 和 `vis_bbox` 被 `_find_generated_images` 排除

## 2. area_loading 模式（区域加载异常）

### 渲染器
`AreaLoadingRenderer` (`app/renderers/area_loading.py`)

### 行为

在 UI 组件的区域中心叠加加载/超时/错误图标，模拟列表加载超时、图片加载失败等场景。

### 输出产物

| 文件 | 说明 | 是否进入序列 |
|---|---|---|
| `final_{timestamp}.png` | 原图 + 加载图标叠加结果 | **是** |

只有一张 `final_*.png`，无其他中间产物。

### 序列构造

其他非 dialog 模式（area_loading、content_duplicate、text_overlay）的序列构造方式**与 dialog 模式相同**，都遵循 `SequenceRewriter` 的统一逻辑：

```
step_N.jpg        - 原图（异常注入于此页面）
step_N+1_anomaly  - 异常图（基于 N 生成）
step_N+2.jpg      - 恢复（复制 N，模拟异常解除后恢复）
step_N+3.jpg      - 原图 N+1 平移
```

区别仅在于异常图的内容：
- dialog: 原图上叠加弹窗
- area_loading: 原图上在目标区域叠加加载图标/暗化
- content_duplicate: 原图底部追加内容浮层
- text_overlay: 原图局部文字被修改

### 特性

- 纯代码计算尺寸（无 LLM 参与）
- VLM 仅用于提取 APP 视觉风格（识别主色、字号等）
- AI 生成风格匹配的图标
- 合成方式：直接在原图上 `paste` 图标 + 区域暗化

## 3. content_duplicate 模式（内容重复异常）

### 渲染器
`ContentDuplicateRenderer` (`app/renderers/content_duplicate.py`)

### 行为

在截图底部生成一个浮层，显示与页面已有组件相同或扩展的内容，模拟"UI 元素重复显示"场景。

### 输出产物

| 文件 | 说明 | 是否进入序列 |
|---|---|---|
| `final_{timestamp}.png` | 原图 + 底部浮层合成结果 | **是** |

只有一张 `final_*.png`。

### 两种子模式

| 模式 | 说明 | 依赖 |
|---|---|---|
| `simple_crop` | 直接裁剪原组件放入底部浮层 | 无 AI 依赖 |
| `expanded_view` | AI 生成扩展视图（如 6 集 → 18 集网格） | VLM + DashScope |

### 序列构造

同统一逻辑。浮层叠加到底部，原图上半部分不变。

### 特性

- 查找目标组件：通过指令关键词匹配 UI-JSON 组件文本
- 浮层尺寸：40%~60% 屏幕高度
- 底部遮罩：变暗效果
- 合成方式：在截图上添加底部色块 + 裁剪/生成的组件内容

## 4. text_overlay 模式（文字编辑异常）

### 渲染器
`TextOverlayRenderer` (`app/renderers/text_overlay.py`)

### 行为

在 App UI 截图上进行精确的局部文字编辑，支持多种操作类型。

### 输出产物

| 文件 | 说明 | 是否进入序列 |
|---|---|---|
| `final_{timestamp}.png` | 编辑后的结果图 | **是** |
| `diff_{timestamp}.png` | 编辑前后差异可视化（红色标注修改区域） | 否（调试用） |
| `edit_plan_{timestamp}.json` | VLM 规划的编辑操作列表 | 否（调试用） |
| `debug_component_crops_{ts}/` | 组件裁剪调试图 | 否（调试用） |

### 操作类型

| 操作 | 说明 | 典型场景 |
|---|---|---|
| `expand_card` | 撑高卡片底部插入文字，后续内容下移 | 在卡片中插入优惠信息 |
| `insert_text` | 在卡片空白处插入文字行 | 补充说明文字 |
| `replace_region` | 替换整个矩形区域内容 | Banner 替换 |
| `modify_text` | 原地替换已有文字 | 修改价格 / 标题 |
| `add_badge` | 添加角标/标签 | "限时优惠"角标 |

### 变体模式

| mode | 说明 |
|---|---|
| `text_overlay` | VLM 规划 → PIL 绘制文字（默认） |
| `modify_text` | 同 `modify_text_ocr` |
| `modify_text_ai` | AI 图像编辑替换文字 |
| `modify_text_ocr` | OCR 精确定位 → PIL 渲染替换 |
| `modify_text_e2e` | 端到端全图 AI 编辑（跳过检测分组） |

### 序列构造

同统一逻辑。注意：`modify_text_e2e` 模式跳过 Stage 1/2（OmniParser 检测 + VLM 分组），直接在全图上做编辑。

### 特性

- 局部编辑：所有操作限定在 bounding box 内，区域外像素 bit-identical
- 风格采样：从原图同区域已有文字采样字号、颜色、粗细
- 使用 PIL + TrueType 字体直接绘制，不使用 AI 图像生成文字
- VLM 仅做编辑规划（"编辑什么"），PIL 执行绘制（"怎么画"）

## 5. 统一序列构造逻辑（SequenceRewriter）

所有模式共用 `SequenceRewriter.rewrite()` 的统一序列逻辑：

```python
# Step 1: 复制全部原始截图
for i in range(len(original_screenshots)):
    shutil.copy2(original_screenshots[i], sequence_dir / f"step_{i:02d}.jpg")

# Step 2: 调用 run_pipeline 生成异常图 → anomaly_generated/final_*.png
# Step 3: 序列重排
#   3a. 注入点后的原图号 +2（腾出两步位置）
#   3b. 注入异常图（step_N+1_anomaly）
#   3c. 复制注入点原图为恢复图（step_N+2）
```

### 最终序列结构

```
modified_sequence/
├── step_00.jpg              原图
├── step_01.jpg              原图
├── step_N.jpg               原图（注入点，异常将出现于此）
├── step_N+1_anomaly.png     异常图（渲染器生成的结果）
├── step_N+2.jpg             恢复图（复制 N，异常解除后回退到此页面）
├── step_N+3.jpg             原图 N+1 平移
├── step_N+4.jpg             原图 N+2 平移
└── ...
```

### `_find_generated_images` 过滤规则

从 `anomaly_generated/` 目录查找异常图时，只取 `final_*.png`，排除：

| 排除模式 | 来源模式 | 原因 |
|---|---|---|
| `dialog_only_*.png` | dialog | 单独弹窗，非合成结果 |
| `vis_bbox_*.png` | dialog | 检测框可视化，调试用 |
| `stage*_*.png` | 所有 | Stage 1/2 中间结果 |
| `diff_*.png` | text_overlay | 差异对比图，调试用 |
| `debug_*` | text_overlay | 组件裁剪调试 |
| `edit_plan_*.json` | text_overlay | JSON 规划，非图片 |
| `*annotated*` | 所有 | 标注可视化 |
| `*mask*` | 所有 | 遮罩调试图 |
